# توثيق نظام تسجيل الدخول وصلاحيات المستخدمين (Authentication & RBAC System)

تم بناء نظام توثيق الهوية وصلاحيات الوصول للوحة التحكم بالاعتماد على قاعدة بيانات **SQL Server 2019** ونظام الصلاحيات المبني على الأدوار (Role-Based Access Control - RBAC).

---

## 1. هيكل قاعدة البيانات (Database Schema)

تتألف قاعدة البيانات من 5 جداول رئيسية تدعم العلاقات متعدد-إلى-متعدد (Many-to-Many) للربط بين المستخدمين، الأدوار، والصلاحيات التفصيلية.

### أ. جدول المستخدمين (`dbo.Users`)
يخزن حسابات مدراء ومحرري لوحة التحكم الخلفية (Backoffice Users).
- **الحقول**:
  - `Id` (BIGINT, IDENTITY, PK): المعرف الفريد للمستخدم.
  - `Username` (NVARCHAR(150), Unique Index): اسم المستخدم الفريد لتسجيل الدخول.
  - `Email` (NVARCHAR(256), Unique Index): البريد الإلكتروني للمستخدم.
  - `PasswordHash` (NVARCHAR(500)): كلمة المرور المشفرة (Hashed).
  - `FullName` (NVARCHAR(250), Nullable): الاسم الكامل للمستخدم.
  - `IsActive` (BIT, Default 1): حالة الحساب (نشط / معطل).
  - `CreatedAt` (DATETIME2, Default GETUTCDATE()): تاريخ إنشاء الحساب.
  - `UpdatedAt` (DATETIME2, Nullable): تاريخ تحديث الحساب.

### ب. جدول الأدوار (`dbo.Roles`)
يحدد المسميات الوظيفية والأدوار الأمنية في النظام مع دعم كامل للغتين العربية والإنجليزية.
- **الحقول**:
  - `Id` (BIGINT, IDENTITY, PK): معرف الدور.
  - `Name` (NVARCHAR(150)): اسم الدور بالعربية (مثال: مدير المحتوى).
  - `Name_En` (NVARCHAR(150)): اسم الدور بالإنجليزية (مثال: Content Manager).
  - `Description` (NVARCHAR(500), Nullable): وصف الدور بالعربية.
  - `Description_En` (NVARCHAR(500), Nullable): وصف الدور بالإنجليزية.
  - `IsSystem` (BIT, Default 0): حقل لحماية الأدوار النظامية الأساسية من الحذف العشوائي.
  - `IsActive` (BIT, Default 1): حالة الدور.
  - `CreatedAt` (DATETIME2, Default GETUTCDATE()).
  - `UpdatedAt` (DATETIME2, Nullable).

### ج. جدول الصلاحيات التفصيلية (`dbo.Permissions`)
يخزن رموز العمليات الصلاحية الدقيقة المتاحة لكل نظام فرعي (مثال: إنشاء مقال، رفع وسائط).
- **الحقول**:
  - `Id` (BIGINT, IDENTITY, PK): معرف الصلاحية.
  - `Code` (NVARCHAR(100), Unique Index): رمز الصلاحية الفريد (صيغة `Module.Action` مثل `Content.Create`).
  - `Name` (NVARCHAR(150)): اسم العملية الصلاحية بالعربية.
  - `Name_En` (NVARCHAR(150)): اسم العملية الصلاحية بالإنجليزية.
  - `Description` / `Description_En` (Nullable): وصف الصلاحية باللغتين.
  - `CreatedAt` (DATETIME2, Default GETUTCDATE()).

### د. الجداول الوسيطة (Many-to-Many Join Tables)
- **جدول أدوار المستخدمين (`dbo.UserRoles`)**:
  - يربط المستخدمين بالأدوار الممنوحة لهم.
  - المفتاح الأساسي المركب: `(UserId, RoleId)`.
  - قيود التكامل المرجعي: تتضمن حذفاً متتابعاً `ON DELETE CASCADE` عند حذف المستخدم أو الدور.
- **جدول صلاحيات الأدوار (`dbo.RolePermissions`)**:
  - يربط الأدوار الفنية بقائمة الصلاحيات المسموح بها.
  - المفتاح الأساسي المركب: `(RoleId, PermissionId)`.
  - قيود التكامل المرجعي: تتضمن `ON DELETE CASCADE`.

## 2. الأدوار والصلاحيات المهيأة افتراضياً (Seeded Data)

تم تهيئة النظام افتراضياً بمجموعة من الأدوار المحددة مسبقاً لحماية تكامل النظام الأساسي:

1. **مدير النظام العام (Super Administrator)**: يمتلك كافة صلاحيات النظام بشكل تلقائي وتمنح له صلاحية الإدارة الشاملة.
2. **مدير النظام (Administrator)**: يمتلك صلاحيات إدارة المحتوى والمستخدمين وإعدادات الموقع الأساسية.
3. **مدير المحتوى (Content Manager)**: مسؤول عن مراجعة وجدولة وإدارة الأقسام والوسائط.
4. **المحرر (Editor)**: مسؤول عن مراجعة وتعديل ونشر مقالات الكتاب والناشرين.
5. **الكاتب (Author)**: يمتلك صلاحية كتابة وتعديل مقالاته الخاصة وحفظها كمسودات.
6. **المترجم (Translator)**: مسؤول عن ترجمة المقالات والوسوم والتصنيفات بين لغات الموقع.
7. **مدير الوسائط (Media Manager)**: مسؤول عن رفع وإدارة ملفات الوسائط المتعددة والصور.
8. **أخصائي سيو SEO (SEO Specialist)**: مسؤول عن إدارة الكلمات المفتاحية وأوصاف الميتا لرفع كفاءة الأرشفة.
